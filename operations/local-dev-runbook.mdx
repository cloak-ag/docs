This runbook is based on commands currently defined in this repo (`programs/justfile`, `services/justfile`, `packages/justfile`, `sdk/package.json`, `web/package.json`).

## 1) Build circuits

From `packages/`:

```bash
just build
```

This builds transaction circuit artifacts and copies them to:

- `web/public/circuits`
- `programs/vkeys/transaction.vkey.bin`

## 2) Deploy/init shield-pool locally

From `programs/`:

```bash
just setup
```

`just setup` builds program, deploys, and initializes pool PDAs.

## 3) Start relay + Postgres

From `services/`:

```bash
just dev
```

This starts local Postgres via docker compose and runs relay.

Minimum envs to verify in `services/.env`:

- `SOLANA_RPC_URL`
- `SOLANA_WS_URL`
- `CLOAK_PROGRAM_ID`
- `DATABASE_URL`

Optional but recommended for local/surfpool setups:

- `COMMITMENT_SYNC_RPC_URL` (separate RPC for sync/scanning reliability)

## 4) Run SDK verification examples

From `sdk/`:

```bash
npm install
npm run build
npx tsx examples/compliance-report-simple.ts
npx tsx examples/scan-history.ts
```

Useful envs:

- `CLOAK_RELAY_URL` (default `http://127.0.0.1:5500`)
- `SOLANA_RPC_URL`
- `CLOAK_PROGRAM_ID`
- `CIRCUITS_PATH` (often `../web/public/circuits`)

## 5) Run web app

From `web/`:

```bash
yarn install
yarn dev
```

Key envs:

- `NEXT_PUBLIC_RELAY_URL`
- `NEXT_PUBLIC_PROGRAM_ID`
- `NEXT_PUBLIC_CIRCUITS_PATH` (typically `/circuits`)

## 6) Verify runtime health

```bash
curl http://127.0.0.1:5500/health
curl http://127.0.0.1:5500/merkle-root
curl http://127.0.0.1:5500/commitments
```

In web privacy history:

- run a deposit/withdraw/swap
- confirm history scan succeeds
- test `Clear cache & rescan`

## Surfpool notes

- Surfpool-style RPCs can be flaky for historical fetches.
- If relay sync is unstable, set `COMMITMENT_SYNC_RPC_URL` to a more reliable/archive RPC while keeping submit RPC unchanged.
- For repeated local tests without resets, use SDK option/env enabling unique nullifier salts.

## Troubleshooting

- `Invalid public inputs size`: use current 264-byte layout.
- `RootNotFound` (`0x1001`): expected under churn; allow SDK retry/regeneration.
- `0x1020` double spend during repeated tests: enable unique nullifiers or reset state.
- `viewing key not found` in history: perform a transaction with viewing-key registration enabled, then rescan.
- `GET /risk-quote` returns 501: configure external quote service or provide `getRiskQuoteInstruction`.
